# контекст проекта

это не корпоративный продукт. это инструмент, написанный для конкретной задачи — поймать билеты на TI 2026 раньше всех. всё в проекте подчинено этой задаче. никакой воды, никаких абстракций ради абстракций, никакого кода впрок.

---

## стилевой язык

у проекта есть свой визуальный язык. он используется везде — в сообщениях бота, в коде, в коммитах, в документации. будущий агент обязан его знать и применять.

**три маркера:**
```
·   (U+00B7)  — ок, норм, готово, спокойно
×   (U+00D7)  — ошибка, что-то не так
▸   (U+25B8)  — алерт, событие, что-то нашли
```

**правила:**
- всё строчными. заголовки, фразы, имена команд — строчными.
- никаких emoji-спама. если emoji — то один и осмысленный. `🚨` допустим для алертов.
- никаких `✅ Успешно!`, `❌ Ошибка!`, `⏳ Подождите...` — это шаблонный ии-паттерн, запрещён.
- `<b>` для заголовков, `<i>` для переходных состояний, `<code>` для timestamp и технических строк.
- HTML parse mode в telegram, не markdown.
- короткие фразы. если умещается в одно слово — одно слово.

**примеры правильного:**
```
· подписан

<i>уведомлю когда появятся билеты на TI 2026</i>
```
```
× ошибка · попробуй позже
```
```
▸ axs — 1 новых · уведомления отправлены
```
```
× <b>axs</b>
<code>failed to fetch: connection refused</code>
```
```
· <b>axs</b> — восстановлен
```

**примеры запрещённого:**
```
✅ Вы успешно подписались на уведомления!
❌ Произошла ошибка. Пожалуйста, попробуйте позже.
⏳ Автоматическая проверка уже выполняется...
```

## голос проекта

прямой. без воды. без корпоративщины. без пояснений очевидного.

если можно сказать в одном слове — одно слово. если нужно объяснить — объясни коротко и точно, как человек который знает о чём говорит, а не как документация энтерпрайз-продукта.

коммит-сообщения тоже в этом стиле. строчными, на русском, по делу:
```
· московское время во всех timestamp
readme · смотрим раньше всех
добавил queue-it детекцию как 4й сигнал
```

## архитектурные принципы

**никакой сложности ради сложности.** три похожих строки лучше абстракции. функция для одного использования не нужна. helper ради одного места не нужен.

**никакой валидации внутреннего кода.** validate только на границах системы — env vars, внешние API. внутри доверяем себе.

**никаких фичей впрок.** не добавляем то, что "может пригодиться". добавляем то, что нужно сейчас.

**error handling без паники.** ошибка монитора не роняет бота. логируем, уведомляем админа, продолжаем работать.

## технологический стек и почему

```
Go           — статический бинарь, FROM scratch, 15MB образ
bbolt        — embedded kv. задача принадлежности множеству. zero infra.
tls-client   — chrome tls fingerprint. cloudflare не видит бота.
flaresolverr — последний эшелон. headless chrome для js-челленджей.
telegram bot — go-telegram-bot-api/v5. polling, не webhooks.
```

не меняй стек без причины. каждый выбор сделан под конкретное ограничение.

## детекция билетов

три слоя на AXS + один независимый сигнал:

```
axs.com ──▸  __NEXT_DATA__ × 3 sources  ──┐
             queue-it waitroom detect      ├──▸  bbolt dedup  ──▸  telegram
steam ────▸  keyword match  ───────────────┘
```

**AXS — три источника в одном JSON:**
- `performerEventsData.eventItems[]` — публичные листинги
- `teamUpcomingEventData.upcomingEvent` — предстоящее событие
- `discoveryPerformerData.events[]` — индекс поиска

**Queue-it** — если AXS включает очередь (`queueit-overlay`, `inqueue.queue-it.net`) — это синтетический сигнал: страница под нагрузкой, билеты живые.

**Steam News** — Valve анонсирует тикетинг через Steam News API (appid 570) раньше чем открывает продажу. самый ранний возможный сигнал.

дедупликация через bbolt: event ID попал в базу → никогда больше не триггернёт.

## время

московское (Europe/Moscow, UTC+3) везде где время отображается пользователю. переменная `moscow *time.Location` объявлена в `cmd/bot/main.go`. при форматировании: `time.Now().In(moscow).Format(...)`.

## контекст события

TI 2026 — The International, ежегодный главный турнир по Dota 2.
- площадка: Oriental Sports Center, Shanghai
- групповая стадия: 13–16 августа 2026
- основной турнир: 20–23 августа 2026
- билеты продаются через AXS (axs.com/teams/1119906/...)
- Valve объявит тикетинг через Steam News раньше чем AXS откроет продажу

бот мониторит оба источника. когда что-то появится — уведомит подписчиков немедленно.

## что не трогать без причины

- `internal/monitor/axs.go` — три источника и queue-it детекция отлажены на реальном трафике
- `internal/fetcher/` — tls fingerprint настроен, не упрощай до стандартного `net/http`
- `docker-compose.yml` — `FLARESOLVERR_URL=http://flaresolverr:8191` в env бота обязателен
- `.gitignore` — `.env` и `data/` не коммитим никогда
